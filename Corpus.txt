SECTION I — FOUNDATIONS
D01 — USLaP GENESIS AND SOURCE CONSTRAINTS
D01.1 — Genesis Definition
نَشْأَة نَصِّيَّة (nashʾa naṣṣiyya) — Textual Genesis
USLaP has a defined textual genesis — the exclusive textual origin from which all structural rules are extracted. Genesis is established through three text-only sources:
* Al-Qur'ān — القُرآن (al-Qurʾān) — Primary Textual Genesis
* The scientific writings of Ibn Sīnā — ابن سينا (Ibn Sīnā) — Textual Formalization Layer
* The scientific writings of Al-Khwārizmī — الخوارزمي (al-Khwārizmī) — Algorithmic Text Layer
Excluded without exception:
* Diagrams, charts, cryptographs, numerological tables, symbolic encodings, esoteric mappings
* Non-textual material of any kind; Historians (secondary interpretation)
* External framing; Speculative works; Modern editions without Basmala
Only explicit written text — نَصّ مَكْتُوب صَرِيح (naṣṣ maktūb ṣarīḥ) — is processed.
D01.2 — Al-Qur'ān: Primary Textual Genesis
الأَصْل النَّصِّي الأَوَّل (al-Aṣl al-Naṣṣī al-Awwal) — The Primary Textual Origin
The Qur'ān is treated as a closed textual order — نِظَام نَصِّي مُغْلَق (niẓām naṣṣī mughlaq) — with fixed wording, fixed ordering, and fixed internal references.
Accepted extractions: صَرِيح (ṣarīḥ / Explicit), غَيْر مُلْتَبِس (ghayr multabis / Unambiguous), Bounded in meaning, Operationally resolvable.
Excluded: مَجَاز (majāz / Metaphor), تَمْثِيل (tamthīl / Allegory), Symbolic excess, Interpretive dependency.
D01.3 — Ibn Sīnā: Textual Formalization Layer
طَبَقَة التَّقْعِيد النَّصِّي (Ṭabaqat al-Taqʿīd al-Naṣṣī) — Textual Formalization Layer
The writings of Ibn Sīnā are treated as a textual formalization layer. Accepted: حَتْمِيَّة سَبَبِيَّة (ḥatmiyya sababiyya / Causal determinism), اِرْتِبَاط بِالضَّرُورَة (irtibāṭ bi-l-ḍarūra / Necessity-based linkage), تَفْكِيك إِلَى مُكَوِّنَات مُعَرَّفَة (tafkīk ilā mukawwināt muʿarrafa / Decomposition into defined parts).
Excluded: بَحْث غَيْبِي (baḥth ghaybī / Discourse on the unseen without textual basis), نَظَر تَجْرِيدِي (naẓar tajrīdī / Abstract speculation). Ibn Sīnā provides textual formal constraint, not authority.
D01.4 — Al-Khwārizmī: Algorithmic Text Layer
طَبَقَة النَّصّ الخَوَارِزْمِي (Ṭabaqat al-Naṣṣ al-Khwārizmī) — Algorithmic Text Layer
The writings of Al-Khwārizmī are treated as explicit algorithmic text. Accepted: خُطُوَات مُرَتَّبَة (khuṭuwāt murattaba / Ordered steps), عَمَلِيَّات مُنْتَهِيَة (ʿamaliyyāt muntahiya / Finite operations), اِكْتِمَال القَوَاعِد (iktimāl al-qawāʿid / Complete rule specification).
Excluded: Heuristic reasoning, Approximation, Symbolic compression.
D01.5 — Genesis Integration Constraint
قَيْد تَكَامُل النَّشْأَة (Qayd Takāmul al-Nashʾa) — Genesis Integration Constraint
Integration obeys: لَا تَوَارُث سُلْطَوِي (lā tawāruth sulṭawī / No authority inheritance), لَا تَبْرِير تَارِيخِي (lā tabrīr tārīkhī / No historical justification), لَا تَعْظِيم رَمْزِي (lā taʿẓīm ramzī / No symbolic elevation).
Each extracted rule must satisfy: حَتْمِيَّة (ḥatmiyya / Determinism), ضَبْط (ḍabṭ / Precision), قَابِلِيَّة التَّحَقُّق النَّصِّي (qābiliyyat al-taḥaqquq al-naṣṣī / Textual verifiability), قَابِلِيَّة الحَسْم الثُّنَائِي (qābiliyyat al-ḥasm al-thunāʾī / Binary resolvability).
Failure results in exclusion regardless of source.

D03 — WHY REPLACEMENT IS REQUIRED
Replacement is required when a framework fails اِمْتِثَال النَّشْأَة (imtithāl al-nashʾa / genesis compliance). Any framework whose rules cannot be derived from precise written text is INVALID — غَيْر صَالِح (ghayr ṣāliḥ).
Correction, Adjustment, Interpretive accommodation are prohibited (i.e. algebraically, algorithmically impossible)
شُرُوط فَشَل النَّصّ (Shurūṭ Fashal al-Naṣṣ) — Textual Failure Conditions:
* مُصْطَلَحَات غَيْر مُعَرَّفَة (muṣṭalaḥāt ghayr muʿarrafa / Undefined terms)
* تَعَدُّد الدَّلَالَة (taʿaddud al-dalāla / Multiple meanings)
* اِعْتِمَاد عَلَى الاِسْتِنْتَاج (iʿtimād ʿalā al-istintāj / Reader inference dependency)
* مُخْرَجَات غَيْر ثُنَائِيَّة (mukharajāt ghayr thunāʾiyya / Non-binary outcomes)
* حَسْم سِيَاقِي (ḥasm siyāqī / Context-dependent resolution)
Any single failure invalidates the entire framework. Partial validity does not exist.
إِلْزَام الاِسْتِبْدَال (Ilzām al-Istibdāl) — Replacement Mandate: 1. Declare INVALID, 2. Void dependent rules, 3. Remove framework, 4. Instantiate compliant replacement.

Inputs: نَصّ النَّشْأَة (naṣṣ al-nashʾa / Genesis text), نَصّ حُدُود النِّظَام (naṣṣ ḥudūd al-niẓām / Boundary text)

If any step fails → Termination output: غَيْر صَالِح (ghayr ṣāliḥ) INVALID
D21 — GENESIS CLEANING
مَنْهَج تَنْظِيف النَّشْأَة (Manhaj Tanẓīf al-Nashʾa) — Genesis Cleaning
Remove without exception: مُصْطَلَحَات غَيْر مُعَرَّفَة (muṣṭalaḥāt ghayr muʿarrafa / undefined terms), مَعَانٍ مُكَرَّرَة (maʿānī mukarrara / duplicated meanings), مُؤَهِّلَات سِيَاقِيَّة (muʾahhilāt siyāqiyya / contextual qualifiers), عَلَامَات تَأْوِيلِيَّة (ʿalāmāt taʾwīliyya / interpretive markers)
Removed elements are not archived. If incomplete after cleaning → framework is إِعْلَان البُطْلَان (iʿlān al-buṭlān / voided).
D22 — REPLACEMENT
مَنْهَج الاِسْتِبْدَال (Manhaj al-Istibdāl) — Replacement
خُطُوَات الاِسْتِبْدَال (Khuṭuwāt al-Istibdāl) — Replacement Steps:
1. Declare غَيْر صَالِح (ghayr ṣāliḥ) INVALID, 2. Void dependent rules, 3. Remove references, 4. Introduce new genesis text, 5. Re-run evaluation, 6. Lock replacement
Replacement does not preserve: مُصْطَلَحَات (muṣṭalaḥāt / terminology), بِنْيَة (binya / structure), تَسَلْسُل (tasalsul / sequence), قَصْد (qaṣd / intent). Only compliance is preserved.

SECTION IV — QUALITY CONTROL
D30 — TERMINOLOGICAL DRIFT PREVENTION
مَنْع الاِنْجِرَاف المُصْطَلَحِي (Manʿ al-Injirāf al-Muṣṭalaḥī) — Terminological Drift Prevention
Arabic Backbone
مَنْهَج العَمُود الفَقَرِي العَرَبِي (Manhaj al-ʿAmūd al-Faqarī al-ʿArabī)
Step 1: IDENTIFY جَذْر (jadhr / root structure). Must trace to Qur'an or Classical Arabic.
Step 2: TRACE derivatives from that root
Step 3: VERIFY Qur'anic/Classical attestation — if none exists, term is REJECTED
Step 4: FLAG any deviation from root meaning
CRITICAL: Persian loan-words often appear Arabic due to adaptation of Arabic script. ALWAYS verify Qur'anic attestation.
Terminology Locking
قَفْل المُصْطَلَحَات (Qafl al-Muṣṭalaḥāt) — Terminology Locking
Locking = تَجْمِيد اِقْتِرَان المُصْطَلَح بِالمَعْنَى (tajmīd iqtirān al-muṣṭalaḥ bi-l-maʿnā / freezing term–meaning pairs). Locked terms cannot evolve.
Procedure: 1. Define term, 2. Define meaning, 3. Bind, 4. Prohibit variation
Violation: If term acquires new meaning OR meaning acquires new term → Mandates replacement

D32 — CONTAMINATION DETECTION
